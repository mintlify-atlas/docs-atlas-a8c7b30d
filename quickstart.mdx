---
title: "Quickstart"
description: "Build and load your first FrankerFaceZ add-on in under 10 minutes."
---

This guide walks you through creating a minimal working add-on — one that registers itself with FFZ, defines a setting, and logs a message when enabled. By the end you'll have a working add-on running in your browser on Twitch.

## Prerequisites

- Node.js 18+ and [pnpm](https://pnpm.io/) installed
- FrankerFaceZ installed in your browser (extension or userscript)
- The FFZ dev server running (see [Installation](/installation))

---

<Steps>

<Step title="Create your add-on directory">

```bash
mkdir my-ffz-addon
cd my-ffz-addon
pnpm init
```

Install FFZ as an external reference (it's a runtime global, not bundled):

```bash
pnpm add --save-dev webpack webpack-cli
```

</Step>

<Step title="Write your add-on class">

Create `src/index.js`:

```javascript src/index.js
import { Addon } from 'utilities/addon';

class MyAddon extends Addon {
  static info = {
    id: 'my-addon',
    name: 'My Add-on',
    author: 'YourName',
    description: 'A minimal FrankerFaceZ add-on.',
    version: '1.0.0'
  };

  async onEnable() {
    this.log.info('My add-on is enabled!');

    // Register a setting
    this.settings.add('my-addon.show-greeting', {
      default: true,
      ui: {
        path: 'Add-Ons > My Add-on >> General',
        title: 'Show greeting in console',
        description: 'Log a greeting message when you open Twitch.',
        component: 'setting-check-box'
      }
    });

    if (this.settings.get('my-addon.show-greeting')) {
      this.log.info('Hello from My Add-on!');
    }

    // Listen for setting changes
    this.settings.on(':changed:my-addon.show-greeting', (value) => {
      this.log.info(`Show greeting changed to: ${value}`);
    });
  }

  async onDisable() {
    this.log.info('My add-on is disabled.');
  }
}

MyAddon.register();
```

<Note>
`Addon` automatically injects `this.settings` and `this.i18n` for you. You don't need to call `this.inject('settings')` manually.
</Note>

</Step>

<Step title="Configure webpack">

Create `webpack.config.js`:

```javascript webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'my-addon.js',
    library: 'MyAddon',
    libraryTarget: 'umd'
  },
  externals: {
    // FFZ utilities are available at runtime from the FFZ bundle
    'utilities/addon': ['FrankerFaceZ', 'utilities', 'addon'],
    'utilities/module': ['FrankerFaceZ', 'utilities', 'module'],
  },
  mode: 'development',
  devtool: 'source-map'
};
```

Build the add-on:

```bash
npx webpack
```

This produces `dist/my-addon.js`.

</Step>

<Step title="Start the FFZ dev server">

If you haven't already, set up and start the FFZ development server:

```bash
# In the FrankerFaceZ source directory
pnpm install
pnpm start
```

The dev server runs on `https://localhost:8000`. Accept the self-signed certificate in your browser.

</Step>

<Step title="Load your add-on">

<Tabs>

<Tab title="Via browser console">

Open Twitch in your browser, open DevTools (F12), and paste your compiled add-on:

```javascript
// Load your compiled add-on from a local file server, or paste the content directly
const script = document.createElement('script');
script.src = 'http://localhost:YOUR_PORT/dist/my-addon.js';
document.head.appendChild(script);
```

Or enable FFZ debug mode to use the dev server:

```javascript
localStorage.ffzDebugMode = true;
location.reload();
```

</Tab>

<Tab title="Via FFZ dev server">

Copy your built `dist/my-addon.js` to the FFZ dev server's `dist/` directory, then enable it from the FFZ add-ons panel in the FFZ settings menu.

</Tab>

</Tabs>

</Step>

<Step title="Verify it works">

Open the browser console on Twitch. You should see:

```
[FFZ:addon.my-addon] My add-on is enabled!
[FFZ:addon.my-addon] Hello from My Add-on!
```

You can also verify through the FFZ object:

```javascript
// In the browser console on Twitch
window.ffz.resolve('addon.my-addon')
// Returns your addon module instance
```

</Step>

</Steps>

## What just happened?

When you called `MyAddon.register()`, it:

1. Registered the add-on's metadata with the `AddonManager`
2. Created a new module instance at `addon.my-addon` in the FFZ module tree
3. Waited for the user to enable the add-on (or enabled it automatically if already in their preferences)
4. Called `onEnable()` once the module and all dependencies were ready
