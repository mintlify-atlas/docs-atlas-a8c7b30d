The CSS tweaks system lives in `src/sites/twitch-twilight/modules/css_tweaks/`. It provides a settings-driven mechanism to inject, toggle, and update CSS rules on the Twitch page without reloading.

## Architecture

`CSSTweaks` extends `Module` with `should_enable = true`. It owns a single `ManagedStyle` instance that holds all dynamically injected styles, and a `chunks` map that tracks which named SCSS chunk files have been loaded.

```js
export default class CSSTweaks extends Module {
	constructor(...args) {
		super(...args);
		this.should_enable = true;
		this.style = new ManagedStyle;
		this.chunks = {};
		this._state = {};
	}
}
```

## The CLASSES map

`CLASSES` is a static object mapping short tweak keys to one or more CSS selectors. These keys are used by `toggleHide()` to inject a `display: none` rule targeting the right element:

```js
const CLASSES = {
	'side-nav': '.side-nav,#sideNav',
	'side-nav-viewers': '.side-nav-card__live-status',
	'prime-offers': '.top-nav__prime',
	'player-ext': '.video-player .extension-taskbar,...',
	'pinned-cheer': '.pinned-cheer,.pinned-cheer-v2,.channel-leaderboard,...',
	'whispers': 'body .whispers-open-threads,...',
	'pinned-hype-chat': '.paid-pinned-chat-message-list',
	// ...
};
```

`toggleHide(key, value)` looks up the selector in `CLASSES` and injects or removes a rule like:

```css
/* generated for key 'prime-offers' when hidden */
.top-nav__prime { display: none !important; }
```

`toggle(key, value)` loads or unloads a named SCSS chunk from the `styles/` directory instead of generating a rule inline.

## Settings integration

Every tweak is backed by a settings entry registered in the `CSSTweaks` constructor. The `changed` callback calls `toggle()` or `toggleHide()` so the style updates immediately when the user changes the setting:

```js
this.settings.add('layout.side-nav.hide-viewers', {
	default: false,
	ui: {
		path: 'Appearance > Layout >> Side Navigation',
		title: 'Hide Channel View Counts',
		component: 'setting-check-box'
	},
	changed: val => this.toggleHide('side-nav-viewers', val)
});
```

## CSS validation

Before injecting any user-provided CSS value (such as a custom font family), FFZ validates it against a throwaway DOM element:

```js
const STYLE_VALIDATOR = document.createElement('span');
```

The validator element is used to test whether a CSS property value is accepted by the browser before applying it to the page.
